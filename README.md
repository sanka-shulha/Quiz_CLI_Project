Консольна вікторина (CLI) — PostgreSQL

Опис проєкту

Консольний застосунок для проходження вікторин з використанням бази
даних PostgreSQL.

Користувач може: - зареєструватися - увійти в систему - проходити
вікторини за категоріями - переглядати свої результати - переглядати
топ-20 - експортувати свої результати у CSV

Адміністратор має окрему панель для керування категоріями та питаннями.

Дані зберігаються в PostgreSQL. Файли використовуються лише для експорту
звітів.

------------------------------------------------------------------------

Використані технології

-   Python 3.13
-   PostgreSQL 18
-   pgAdmin 4
-   psycopg2-binary
-   python-dotenv
-   bcrypt

------------------------------------------------------------------------

Структура бази даних

users
-   id
-   login (UNIQUE)
-   password (bcrypt hash)
-   birth_date
-   is_admin (BOOLEAN)

categories
-   id
-   name (UNIQUE)
-   is_active (BOOLEAN)
questions

-   id
-   category_id (FK → categories)
-   text
-   option_a
-   option_b
-   option_c
-   option_d
-   correct_option (a/b/c/d)

results
-   id
-   user_id (FK → users)
-   category_id (FK → categories)
-   score
-   total_questions
-   created_at
------------------------------------------------------------------------
Реалізована функціональність

Основні можливості
-   Реєстрація користувача (паролі зберігаються у вигляді bcrypt-хешу)
-   Вхід у систему
-   Старт вікторини за категорією
-   Перевірка валідності відповіді (програма перепитує, поки не введено
    a/b/c/d)
-   Перевірка: якщо в категорії менше 20 питань — виводиться
    повідомлення
-   Збереження результату в БД
-   Перегляд власних результатів
-   Топ-20 результатів (тільки по активних категоріях)
-   Зміна пароля
-   Зміна дати народження
-   Експорт результатів у CSV
------------------------------------------------------------------------

Адміністратор
-   поле is_admin
-   окрема адмін-панель
-   CRUD для категорій
-   CRUD для питань
-   Safe delete категорій (категорія деактивується, без каскадного
    видалення)
-   Неактивні категорії не відображаються звичайним користувачам
------------------------------------------------------------------------

Обробка помилок
Усі CRUD-операції обгорнуті в try/except з rollback.
------------------------------------------------------------------------

Ініціалізація бази даних
1.  Створити БД у pgAdmin з назвою: quiz_shulha
2.  Виконати файл schema.sql
3.  Виконати файл seed.sql
------------------------------------------------------------------------

Налаштування .env - створено файл .env у корені проєкту:

DB_NAME=quiz_shulha DB_USER=postgres DB_PASSWORD=your_password
DB_HOST=localhost DB_PORT=5432
------------------------------------------------------------------------

Встановлення залежностей
py -m pip install -r requirements.txt
requirements.txt містить: - psycopg2-binary - python-dotenv - bcrypt

------------------------------------------------------------------------

Запуск програми py main.py

------------------------------------------------------------------------

Демонстраційний сценарій

1.  Зареєструвати користувача
2.  Увійти
3.  Пройти вікторину
4.  Переглянути свої результати
5.  Переглянути топ-20
6.  Експортувати результати
7.  Увійти як адміністратор
8.  Деактивувати категорію
9.  Переконатися, що вона не відображається для звичайного користувача
